| **분류** | **사용자/상황 시나리오** |
| --- | --- |
| 가용 가능 시간 관리 | 사용자는 주/월 단위로 가용시간과 버퍼를 설정하고, 그 제약을 넘지 않도록 계획 총량을 통제할 수 있다. |
| 프로젝트(목표) 관리 | 사용자는 여러 프로젝트를 동시에 운영하면서, 프로젝트별로 투입률(시간 배분 비율/총량)을 확인하고 필요 시 조정할 수 있다. |
|  | 사용자가 프로젝트의 투입률을 변경하면, 시스템은 가용시간·버퍼 제약을 고려해 조정이 필요한 일정/작업 범위를 드러내거나 재배치 근거를 제공할 수 있다. |
| 일정(테스크) 관리 | 사용자는 프로젝트에 속한 일정을 가용시간과 버퍼를 고려하여 달력의 시간 구간(또는 실행 순서)으로 배치할 수 있다. |
|  | 계획이 틀어지면 사용자는 일정을 미루거나(시간 이동), 순서를 바꾸거나, 수행 여부를 조정할 수 있고, 시스템은 그 조정이 가용시간·버퍼·우선순위·기한과 충돌하는지 보여줄 수 있다. |
| 집중 지원(포모도로) | 사용자는 캘린더에서 특정 일정(작업)을 선택해 집중 지원을 시작할 수 있고, 시스템은 해당 일정과 연결된 포모도로 타이머를 제공해야 한다. |
|  | 사용자가 포모도로를 수행하면, 시스템은 해당 일정의 실제 수행 시간/세션 결과를 기록하여 이후 계획 및 조정의 입력으로 활용할 수 있다. |
| 회고/학습 | 사용자는 계획 대비 실제 수행의 차이를 확인하고, 그 결과가 다음 주/월 계획(가용시간, 버퍼, 투입률, 일정 배치)에 반영되도록 할 수 있다. |
| 타임라인 | 사용자는 프로젝트별로 해야 할 일의 구조(작업 목록/분해)를 타임라인에서 확인하고, 캘린더에서는 그 작업들이 언제 배치되었는지(시간 구간)를 확인할 수 있다. |
| 대시보드 | 사용자는 프로젝트별 투입률, 가용시간 대비 계획 총량, 버퍼 소진, 일정 완료율/수행률을 한 화면에서 확인하고 다음 조정(투입률/범위/배치)을 결정할 수 있다. |
| 긴급 테스크 처리 | 사용자는 계획에 없던 긴급 업무가 발생하면, 캘린더에서 빠르게 테스크를 생성하고 배치할 수 있으며, 시스템은 이로 인해 영향받는 기존 일정을 보여주고 재배치 대안을 제안할 수 있다. |
| 반복 일정 관리 | 사용자는 매일/매주/매월 반복되는 일정(정기 미팅, 루틴 등)을 한 번에 등록하고, 시스템은 해당 기간 동안 자동으로 일정을 생성하여 가용시간과 버퍼를 반영할 수 있다. |
|  | 사용자가 반복 일정 중 하나를 수정하면, 시스템은 "이 일정만 수정" 또는 "앞으로 모두 수정" 옵션을 제공하여 의도를 명확히 할 수 있다. |
| 알림/리마인더 | 사용자는 마감 임박 테스크(1일 전, 당일)에 대한 알림을 받아 놓치지 않을 수 있다. |
|  | 사용자는 전역 버퍼 또는 프로젝트 버퍼가 80% 이상 소진되면 조기 경고를 받아 계획 조정을 고려할 수 있다. |
|  | 사용자는 우선순위 높은 테스크가 일정 기간(예: 3일) 동안 미배치 상태이면 알림을 받아 배치를 유도받을 수 있다. |
|  | 사용자는 주 시작 시(월요일 아침) 이번 주 계획 검토 리마인더를 받아 계획을 점검할 수 있다. |
| 온보딩 | 신규 사용자는 첫 로그인 시 4단계 마법사를 통해 가용시간, 버퍼, 프로젝트, 첫 일정 배치를 체험하며 서비스 핵심 개념을 학습할 수 있다. |
|  | 사용자는 온보딩 단계를 건너뛸 수 있지만, 최소 1개의 프로젝트는 생성되어야 하며, 건너뛴 설정은 대시보드에 배너로 재알림된다. |
