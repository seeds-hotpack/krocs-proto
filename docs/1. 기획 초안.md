# 1. 서비스 가치 정의

> 사용자가 목표와 일정을 현실성 있게 계획하고 실행력 있게 수행하며 변화에 따라 근거를 기반으로 조정하도록 돕고, 경험이 축적될수록 계획과 조정 의사결정의 정확도를 지속적으로 높인다.
>

# 2. 서비스 목표

- 우선순위·기한·가용 시간 제약을 반영해 실행 가능한 계획을 설계하고, 변경 상황에서는 영향을 최소화하는 대안(일정 재배치 등)과 근거를 제시한다.
- 사용자가 계획된 작업을 수행하는 동안, 집중을 유지할 수 있는 실행 지원 기능(포모도로 타이머 등)을 제공하여 실제 수행률과 완료율을 높인다.
- 실제 수행 데이터를 반영해 작업 소요 시간 예측과 집중 가능 시간 추정치를 지속적으로 보완함으로써, 시간이 지날수록 계획과 변경 의사결정의 정확도를 개선한다.

# 3. 목표 사용자 정의

## 3.1. 파워 플래너 (1차 목표 사용자)

자신의 시간을 주/월 단위로 운영하면서, 여러 프로젝트를 동시에 관리하고, 각 프로젝트에 포함된 일정(해야 할 일)을 1개 이상의 실행 단위로 구체화하여 주/월 계획 안에서 시간 구간 또는 실행 순서로 배치·운영하는 사용자이다.

- 이 사용자는 계획이 틀어질 때 일정의 시간·순서·수행 여부 중 하나 이상을 조정하며, 조정 판단에서 우선순위(비교 가능한 값), 기한(달력 시각), 주/월 가용 가능 시간과 버퍼(총량)를 근거로 삼아 시간을 재배분한다.
- 이 사용자는 계획 대비 실제 수행의 차이를 다음 주/월 계획에 반영하여, 자신의 시간 배정과 시간 사용의 현실성을 점진적으로 높이려는 경향을 가진다.

## 3.2. 성장형 플래너 (2차 목표 사용자)

시간 관리 의지는 있으나 계획이 자주 무너지고, 변경 상황에서 무엇을 먼저 미뤄야 하는지에 대한 기준이 약해 계획 유지가 어려운 사용자이다.

- 이 사용자는 일을 완전히 세분화하지 않더라도, 일정(해야 할 일)에 대해 우선순위 또는 기한 중 하나를 명시하고, 주/월 단위의 가용 가능 시간과 버퍼라는 최소 제약을 유지하는 방식으로 계획을 갱신할 수 있다.
- 이 사용자는 계획 변경이 발생했을 때도 동일한 최소 기준을 따라 일정 조정을 반복함으로써, 계획을 끝까지 유지하는 능력을 점진적으로 형성하려는 성향을 가진다.

## 3.3. 확장 타깃 (향후 확장 사용자층)

개인 시간 운영을 넘어, 동일한 프로젝트 단위를 기준으로 여러 사람이 함께 참여하거나 서로를 기준점으로 삼아 동기부여를 얻으려는 사용자층이다. 이 사용자층은 프로젝트별로 참여 인원을 추가하는 방식의 확장을 전제로 하며, 확장 방향은 협업과 경쟁의 두 관점으로 구분된다.

- **협업 관점**에서 이 사용자층은 동일한 프로젝트에 포함된 일정(해야 할 일)의 분해, 실행 순서, 시간 배정과 같은 계획 요소를 참여자 간에 정렬하고, 변경 상황에서도 동일한 기준으로 조정하여 공동의 완료를 달성하려는 성향을 가진다.
- **경쟁 관점**에서 이 사용자층은 프로젝트 달성 과정에서 개인별 시간 운영과 실행 수준을 비교 가능한 형태로 확인하고, 비교 결과를 동기부여로 사용하여 수행 강도와 지속성을 높이려는 성향을 가진다.

# 4. 온보딩 및 사용자 학습 경로

## 4.1. 초기 설정 마법사 (First-time User Experience)

신규 사용자가 서비스의 핵심 개념을 이해하고 최소한의 설정을 완료할 수 있도록 돕는 단계별 가이드를 제공한다.

### 단계 1: 가용시간 설정
- 사용자에게 주당 계획 가능한 시간을 입력받는다
- 예시를 제공한다: "주 40시간 근무자는 보통 20-30시간, 학생은 30-40시간 정도 계획합니다"
- 월 단위 선호 시 월간 가용시간으로 전환 옵션 제공

### 단계 2: 버퍼 개념 소개
- 버퍼의 목적을 설명한다: "예상치 못한 일이나 계획 변경을 위한 여유 시간"
- 추천값을 제시한다: 가용시간의 20% (예: 주 30시간 계획 시 6시간 버퍼)
- 사용자가 조정 가능하도록 슬라이더 제공

### 단계 3: 첫 프로젝트 생성
- 사용자가 첫 번째 프로젝트를 만들도록 유도한다
- 예시 제공: "업무", "개인 성장", "사이드 프로젝트" 등
- "나중에 추가하기" 옵션 제공

### 단계 4: 첫 일정 배치 체험
- 간단한 테스크 1개를 만들고 캘린더에 배치해보도록 안내
- 버퍼 사용 여부, 가용시간 반영 등을 실시간으로 보여줌
- 완료 시 "시작하기" 버튼으로 메인 화면 진입

### 건너뛰기 정책
- 모든 단계에서 "나중에 하기" 옵션 제공
- 단, 최소 1개의 프로젝트는 생성되어야 서비스 이용 가능
- 건너뛴 설정은 대시보드 상단에 배너로 재알림

## 4.2. 점진적 기능 노출 (Progressive Disclosure)

"성장형 플래너"를 위해 복잡한 기능을 단계적으로 노출한다.

### Level 1: 기본 계획 (첫 1-2주)
- 프로젝트, 테스크, 일정 배치
- 가용시간, 기본 버퍼
- 포모도로 타이머

### Level 2: 조정과 학습 (3-4주차)
- 투입률 개념 소개
- 재배치 기능
- 회고 작성

### Level 3: 최적화 (5주차 이후)
- 프로젝트별 버퍼
- 예측 보정
- 상세 분석

### 진행 트리거
- 특정 액션 횟수 도달 시 자동으로 다음 레벨 기능 소개
- 사용자가 설정에서 "전체 기능 보기" 토글 가능

## 4.3. 컨텍스트 도움말

각 화면에서 "?" 아이콘 클릭 시 해당 기능의 목적과 사용법을 간단히 설명하는 툴팁 또는 사이드 패널을 제공한다.
